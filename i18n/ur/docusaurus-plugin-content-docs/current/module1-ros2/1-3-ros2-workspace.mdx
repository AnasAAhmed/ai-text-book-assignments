# ورک اسپیس اور پیکیجز (Workspace and Packages)

ایک آر او ایس 2 (ROS 2) **ورک اسپیس** آپ کے فائل سسٹم پر ایک ایسی ڈائرکٹری ہے جہاں آپ اپنے ROS 2 پیکیجز کو محفوظ، بلڈ (build) اور مینیج کر سکتے ہیں۔ یہ آپ کی روبوٹ ایپلی کیشنز تیار کرنے کا مرکزی مقام ہے۔



## کولکن: بلڈ سسٹم (Colcon: The Build System)

`colcon` دراصل ROS 2 کا بنیادی بلڈ ٹول ہے۔ یہ ایک کمانڈ لائن ٹول ہے جو ورک اسپیس میں موجود متعدد پیکیجز کو بلڈ کرنے، ٹیسٹ کرنے اور انسٹال کرنے کے عمل کو خودکار بناتا ہے۔ `colcon` کو لچکدار اور قابلِ توسیع (extensible) بنایا گیا ہے تاکہ یہ مختلف بلڈ اقسام (مثلاً `ament_cmake` اور `ament_python`) کو سپورٹ کر سکے۔

## ورک اسپیس بنانا (Creating a Workspace)

ایک نئی ROS 2 ورک اسپیس بنانے کے لیے عام طور پر درج ذیل مراحل پر عمل کیا جاتا ہے:

1. **ورک اسپیس ڈائرکٹری بنائیں:**
   ```bash
   mkdir -p ~/ros2_ws/src
یہ src سب ڈائرکٹری وہ جگہ ہے جہاں آپ کے سورس پیکیجز موجود ہوں گے۔

ورک اسپیس میں داخل ہوں:

Bash

cd ~/ros2_ws
پیکیجز (Packages)
ایک پیکیج ROS 2 میں تنظیم (organization) کی بنیادی اکائی ہے۔ اس میں سورس کوڈ، بلڈ اسکرپٹس، کنفیگریشن فائلز اور دیگر وسائل شامل ہوتے ہیں جو مل کر ایک مخصوص فنکشن فراہم کرتے ہیں (مثلاً کسی سینسر کا ڈرائیور، پاتھ پلاننگ کا الگورتھم، یا لانچ فائلوں کا مجموعہ)۔

ہر ROS 2 پیکیج میں package.xml فائل کا ہونا لازمی ہے، جو پیکیج کے بارے میں میٹا ڈیٹا (نام، ورژن، تفصیل، انحصار/dependencies اور مینٹینرز) فراہم کرتی ہے۔

نیا پیکیج بنانا (Creating a New Package)
آپ ros2 pkg create کمانڈ استعمال کرتے ہوئے ایک نیا ROS 2 پیکیج بنا سکتے ہیں۔ یہ کمانڈ ضروری فائلوں کے ساتھ ایک بنیادی پیکیج ڈھانچہ تیار کر دیتی ہے۔

پائتھن (Python) پیکیج کے لیے:

Bash

cd ~/ros2_ws/src
ros2 pkg create --build-type ament_python my_robot_controller
سی پلس پلس (C++) پیکیج کے لیے:

Bash

cd ~/ros2_ws/src
ros2 pkg create --build-type ament_cmake my_sensor_driver
ورک اسپیس کو بلڈ کرنا (Building a Workspace)
ورک اسپیس میں پیکیجز شامل کرنے یا ان میں ترمیم کرنے کے بعد، آپ کو colcon کے ذریعے انہیں بلڈ کرنے کی ضرورت ہوتی ہے:

Bash

cd ~/ros2_ws
colcon build
یہ کمانڈ آپ کے کوڈ کو کمپائل کرتی ہے، ایگزیکیوٹیبل (executable) فائلیں تیار کرتی ہے، اور ضروری فائلوں کو آپ کی ورک اسپیس کے اندر موجود install ڈائرکٹری میں انسٹال کر دیتی ہے۔

سیٹ اپ فائلوں کی سورسنگ (Sourcing the Setup Files)
اپنی ورک اسپیس سے ایگزیکیوٹیبلز اور لائبریریوں کو استعمال کرنے کے لیے، آپ کو colcon کی تیار کردہ سیٹ اپ فائلوں کو سورس کرنا ہوگا۔ یہ عمل آپ کے ورک اسپیس کے انوائرمنٹ ویری ایبلز کو آپ کے شیل (shell) میں شامل کر دیتا ہے۔

Bash

# اپنی ~/ros2_ws ڈائرکٹری میں
source install/setup.bash  # bash/zsh کے لیے
# source install/setup.ps1 # PowerShell کے لیے
عام طور پر پہلے مین ROS 2 انسٹالیشن کو سورس کیا جاتا ہے اور پھر اپنی ورک اسپیس کی سیٹ اپ فائلوں کو، تاکہ یہ یقینی بنایا جا سکے کہ آپ کے ورک اسپیس کے پیکیجز کو ترجیح ملے۔

اوورلیز (Overlays)
ROS 2 ورک اسپیسز کو اوورلے (overlay) کیا جا سکتا ہے۔ اس کا مطلب ہے کہ آپ کسی موجودہ ورک اسپیس کے اوپر ایک نئی ورک اسپیس بنا سکتے ہیں (مثلاً اپنی مین ROS 2 انسٹالیشن کے اوپر اپنی ڈویلپمنٹ ورک اسپیس)۔ یہ آپ کو پوری ROS 2 ڈسٹری بیوشن کو دوبارہ بلڈ کیے بغیر مخصوص پیکیجز پر کام کرنے کی اجازت دیتا ہے۔

colcon اور پیکیجز پر مبنی یہ ورک اسپیس ماڈل ROS 2 میں ماڈیولرٹی، دوبارہ استعمال کی اہلیت اور کام کے مؤثر بہاؤ کو فروغ دیتا ہے۔
