# آواز سے عمل تک کی پائپ لائن (The Voice-to-Action Pipeline)

ایک وی ایل اے (VLA) سسٹم کا جوہر جو زبانی احکامات پر ردعمل دیتا ہے، اس کی **آواز سے عمل (Voice-to-Action) پائپ لائن** ہے۔ یہ پائپ لائن انسانی آواز کو ٹھوس اور قابلِ عمل روبوٹک کمانڈز میں تبدیل کرتی ہے۔ یہ عام طور پر تین اہم مراحل پر مشتمل ہوتی ہے: آواز کی شناخت (voice recognition)، ارادے کا استخراج (intent extraction)، اور کام کی تقسیم (task breakdown)۔



### وسپر (Whisper): آواز کی شناخت کے لیے

کسی بھی آواز سے کنٹرول ہونے والے سسٹم کا پہلا قدم بولے گئے الفاظ کو متن (text) میں تبدیل کرنا ہے۔ **OpenAI Whisper** ایک بہترین ہمہ گیر ماڈل ہے جو اس کام میں مہارت رکھتا ہے۔ یہ لہجوں، پس منظر کے شور اور تکنیکی زبان کو سمجھنے میں بہت مضبوط ہے، جو اسے روبوٹکس کے لیے انتہائی موزوں بناتا ہے۔

وسپر کو آپ کی مشین پر مقامی طور پر (بشمول جی پی یو پر تیز رفتاری کے لیے) چلایا جا سکتا ہے یا اے پی آئی (API) کے ذریعے رسائی حاصل کی جا سکتی ہے۔ روبوٹکس کے لیے، مقامی اور ریئل ٹائم نفاذ کو زیادہ ترجیح دی جاتی ہے۔

**یہ کیسے کام کرتا ہے (سادہ الفاظ میں)**:
1. **آڈیو ان پٹ**: روبوٹ کا مائیکروفون آواز کو ریکارڈ کرتا ہے۔
2. **پری پروسیسنگ**: آڈیو کو فلٹر کیا جاتا ہے اور وسپر ماڈل کو بھیجا جاتا ہے۔
3. **ٹرانسکرپشن**: وسپر بولے گئے کمانڈ کو ٹیکسٹ اسٹرنگ کی صورت میں آؤٹ پٹ دیتا ہے۔

**کوڈ کی مثال (Python اور whisper لائبریری کے ساتھ)**:

```python
import whisper

def transcribe_audio(audio_file_path):
    """
    OpenAI کے Whisper ماڈل کا استعمال کرتے ہوئے آڈیو فائل کو متن میں تبدیل کرتا ہے۔
    """
    model = whisper.load_model("base") # بہتر درستگی کے لیے "small" یا "medium" بھی استعمال کر سکتے ہیں
    result = model.transcribe(audio_file_path)
    return result["text"]

# استعمال کی مثال:

زبان سے ارادے کا استخراج (Intent Extraction)
جب ہمارے پاس متن کی شکل میں کمانڈ آ جاتی ہے، تو روبوٹ کو یہ سمجھنے کی ضرورت ہوتی ہے کہ صارف کا ارادہ (Intent) کیا ہے۔ اسے 'انٹنٹ ایکسٹریکشن' کہتے ہیں۔ اس میں بنیادی عمل (action)، ملوث اشیاء (objects)، اور کسی بھی قسم کی شرائط کو پہچاننا شامل ہے۔

مثال کے طور پر، اگر کمانڈ ہے "کچن ٹیبل پر جاؤ اور نیلا مگ اٹھاؤ":

ارادہ (Intent): نیویگیٹ کرنا، اٹھانا (Pick Up)

ہدف مقام (Target Location): کچن ٹیبل

ہدف شے (Target Object): مگ

شرائط (Constraints): رنگ (نیلا)

ایل ایل ایم (LLMs) ارادے کو سمجھنے میں غیر معمولی طور پر ماہر ہوتے ہیں۔ آپ ایل ایل ایم کو متن دے کر اس سے ارادے کی ایک منظم (structured) شکل طلب کر سکتے ہیں۔

LLM پرومپٹ کی مثال:

Plaintext

صارف کی کمانڈ: "کچن ٹیبل پر جاؤ اور نیلا مگ اٹھاؤ۔"

روبوٹ کا ارادہ، اعمال، ہدف مقامات اور ہدف اشیاء نکالیں۔ جواب JSON فارمیٹ میں دیں۔

{
  "actions": [
    {"type": "navigate", "location": "kitchen table"},
    {"type": "pick_up", "object": {"name": "mug", "color": "blue"}}
  ]
}
ایل ایل ایم (LLMs) کا استعمال کرتے ہوئے کام کی تقسیم
انسانوں کے بہت سے احکامات تجریدی (abstract) ہوتے ہیں۔ ایک روبوٹ براہ راست "کمرہ صاف کرو" پر عمل نہیں کر سکتا۔ اسے ٹاسک بریک ڈاؤن (Task Breakdown) کی ضرورت ہوتی ہے—یعنی چھوٹے، مخصوص اور قابلِ عمل اقدامات کا ایک سلسلہ۔

ایک اعلیٰ سطحی ارادے کو دیکھتے ہوئے، ایک ایل ایل ایم ذیلی کاموں کا ایک سلسلہ تیار کر سکتا ہے:

تقسیم (Decomposition): ایک بڑے کام کو چھوٹے مراحل میں توڑنا۔

ترتیب (Ordering): ان مراحل کو منطقی ترتیب میں رکھنا۔

اصلاح (Refinement): ہر قدم میں تفصیلات یا شرائط شامل کرنا۔

مثال: "کمرہ صاف کرو" → ایکشن گراف

ایل ایل ایم روبوٹ کی صلاحیتوں اور کمرے کے نقشے کو مدنظر رکھتے ہوئے درج ذیل ٹاسک بریک ڈاؤن تیار کر سکتا ہے:

JSON

{
  "high_level_command": "Clean the room",
  "task_graph": [
    {
      "id": "1",
      "action": "navigate_to_location",
      "parameters": {"location": "living_room_center"}
    },
    {
      "id": "2",
      "action": "identify_and_localize_objects",
      "parameters": {"objects_of_interest": ["cup", "book"]}
    },
    {
      "id": "3",
      "action": "pick_up_object",
      "parameters": {"object_id": "cup_01"}
    }
    // ... مزید اقدامات
  ]
}
یہ منظم ٹاسک بریک ڈاؤن، جو درحقیقت ایک ایکشن گراف ہے، روبوٹ کے کنٹرول سسٹم کے ذریعے نافذ کیا جا سکتا ہے۔ گراف کا ہر "عمل" روبوٹ کی ایک بنیادی صلاحیت کے مطابق ہوتا ہے، جو اکثر ROS 2 ایکشن سرور کے طور پر نافذ کیا جاتا ہے۔
